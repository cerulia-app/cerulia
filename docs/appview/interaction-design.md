# Interaction Design

このファイルは AppView の導線・UI パターン・インタラクションモデルを定義する。
画面ごとのレイアウト仕様は各 wireframe ファイルを正本とし、このファイルは「どこから何クリックで到達できるか」「どの UI が出てくるか」を確定する。

---

## ナビゲーションシェル

### デスクトップ（≥1024px）— 左サイドバー

左サイドバーは常時表示する固定領域（幅 240px）。

```
┌──────────────────────────┐
│  ◇ Cerulia               │  → /home
├──────────────────────────┤
│  ホーム                  │  → /home
│  キャラクター            │  → /profile/[自分のactor]#characters
│  記録                    │  → /profile/[自分のactor]#sessions
│  マイページ              │  → /profile/[自分のactor]
├──────────────────────────┤
│                          │
│   （スペーサー）         │
│                          │
├──────────────────────────┤
│ ┌──────────────┬──────┐  │
│ │  ＋ 作成     │  ∨   │  │  ← スプリットボタン
│ └──────────────┴──────┘  │
└──────────────────────────┘
```

`[＋ 作成]` 左部分のクリック → `/characters/new`（キャラクター作成をデフォルト主動作とする）

`[∨]` 右部分のクリック → ドロップダウンメニューを開く：

```
┌───────────────────────────────┐
│  ✦  セッションを記録          │
│  □  シナリオを登録            │
│  ◎  キャンペーンを作成        │
│  ⌂  ハウスを作成              │
└───────────────────────────────┘
```

ドロップダウン選択後 → 画面右端から右サイドペインが開き、対応する作成フォームが表示される。

```
┌──────────────────────────┐
│ ─────  ×                 │  ← ドラッグハンドル、閉じるボタン
├──────────────────────────┤
│                          │
│   [icon]                 │

│   キャラクターを作成する │
│                          │

│   │    作成する       │   │  ← タップで /characters/new へ

│   ● ○ ○ ○ ○            │  ← ページインジケーター
```

| 5 | ハウス作成 | モーダル閉じ → ボトムシート（ハウス作成フォーム）|

## UI パターン定義

### 右サイドペイン（デスクトップ）

| 属性 | 仕様 |
|------|------|
| 幅 | 380px 固定 |
| 出現 | 右端から slide-in。メインコンテンツ幅は自動縮小 |
| 閉じ方 | ヘッダー `×` ボタン、ペイン外クリック、`Escape` キー |
| 状態 | 詳細表示（デフォルト）↔ 編集モード（「編集する」ボタンでトグル）|
| 保存 | ペイン内の「保存する」ボタンで完結。ページ遷移しない |
| ナビゲーションスタック | ペイン内の関連リンクをクリックすると、新しい内容が同一ペイン内へ右からスライドイン。前画面はスタックに積まれる。最大 3 階層まで積む |

```
┌────────────────────┬───────────────────┐
│   メインコンテンツ │ ←  タイトル    [×]│  ← 戻る矢印（スタックがある時）
│   （縮小）         │─────────────────  │
│                    │  詳細内容         │
│                    │                   │
│                    │  [編集する]       │
│                    │                   │
│                    │  [削除する]       │
└────────────────────┴───────────────────┘
```

**スタック動作の詳細**:

- ペイン左上の `←` は親ノードへ戻る（iOS の navigation controller と同じモデル）
- `×` は常にスタック全体を閉じる
- URL クエリは最前面のペインだけを保持する（例: `?pane=session:abc`）
- ペインスタックはページ内状態として保持する。共有 URL とリロード復元は最前面のペインだけを対象にする
- ブラウザの戻る/進むは最前面ペインの変化に対応する。履歴 entry は最前面ペイン単位で push する
- スタック深度の上限は 3。さらに開く操作は最古を破棄して新しいものを積む
- `Escape` キーは「1 階層戻る」。スタックが 1 階層のときだけペインを閉じる

### モーダルボトムシート（モバイル）

| 属性 | 仕様 |
|------|------|
| 初期高さ | 画面の 50%（ peek 状態）|
| 展開 | 上スワイプで全画面展開 |
| 閉じ方 | 下スワイプ、ハンドル上をタップで peek → close |
| 状態 | 詳細表示（デフォルト）↔ 編集モード（「編集する」タップでトグル）|
| 保存 | シート内の「保存する」ボタン（下部 sticky）で完結 |
| 重ね表示 | シート内の関連リンクをタップすると、新しいシートが既存シートの上に重なって下からスライドアップ。下のシートは僅かに縮小して背景化 |

**重ね表示の詳細**:

```
┌──────────────────────────┐
│  （メインコンテンツ）     │
├──────────────────────────┤  ← 一段下のシートのヘッダーが見える
│  ─── ←  ×                │  ← 最前面シート: 戻る矢印 + 閉じる
├──────────────────────────┤
│                          │
│  詳細内容                │
│                          │
└──────────────────────────┘
```

- 最前面シートの左上に `←`（前のシートへ戻る）と `×`（全シートを閉じる）を併置する
- 下スワイプは「現在のシートだけを閉じる」。一段下のシートが残る
- 下スワイプを peek 状態から続けると全シートが閉じる
- 重ね深度は 3 まで。それ以上は最古を破棄して新しいものを積む
- URL クエリは最前面のシートだけを保持する（例: `?pane=campaign:abc`）
- シートの重なり順はページ内状態として保持する。共有 URL とリロード復元は最前面のシートだけを対象にする
- ブラウザバック → 最前面のシートを閉じる（次のバックでさらに 1 段戻る）

## 導線マップ

### トップ → サインイン → ホーム

```
/ (top)
  └── 「サインインして作成を始める」(primary CTA)
        └── OAuth flow
              └── /home  [2 クリック]
```

### 共有リンク受け取り → 利用開始

```
共有 URL (例: characters/[branch])
  └── /characters/[branch]  [0 クリック: 直接開く]
        └── 「Cerulia で作成を始める」(CTA in footer or banner)
              └── /  [1 クリック]
                    └── 「サインインして作成を始める」
                          └── /home  [2 クリック]
```

合計 3 アクションで共有リンク受け取りから自分の作成ホームに到達できる。

### ペイン内ナビゲーションの具体例

#### セッション詳細 → 関連キャンペーン（デスクトップ）

```
[操作] /profile/[actor] のセッションタブでセッション A をクリック
  → ペイン Stack[0]: セッション A 詳細
[操作] ペイン内の「キャンペーン: B →」リンクをクリック
  → ペイン Stack[1]: キャンペーン B 詳細（スライドイン）
     ヘッダー左に [←] 表示
[操作] [←] クリック → セッション A 詳細に戻る
```

#### キャンペーン詳細 → 紐づくセッション（デスクトップ）

```
[操作] /profile/[actor] のキャンペーンタブでキャンペーン B をクリック
  → ペイン Stack[0]: キャンペーン B 詳細（紐づくセッション一覧を含む）
[操作] 一覧内のセッション C をクリック
  → ペイン Stack[1]: セッション C 詳細（スライドイン）
[操作] セッション C 内の「キャンペーン: B →」をクリック
  → 既に Stack[0] に同 ID が積まれているため、Stack[1] を破棄して Stack[0] にフォーカスを戻す（ループ防止）
```

#### モバイルでの同等動作

- シートが下から積まれる以外は同じ
- 重複検知も同じく行い、循環参照を防ぐ

### 自分のコンテンツを参照・編集

`/profile/[自分のactor]` には 5 つのタブがある。初期タブはキャラクター。
`actor` は DID または handle を受け付ける。handle の例は `/profile/vrc.minetake.net`。AppView は handle を DID に解決し、双方向検証に成功した handle だけを有効な actor alias として扱う。
他人のプロフィール `/profile/[相手のactor]` にも同じタブが並ぶが、表示されるのは public 状態の record だけで、編集導線は出さない。

#### キャラクタータブ

キャラクターは Cerulia の主目的の共有対象であり、独立したページを持つ。

```
/profile/[actor]（キャラクタータブ）
  └── キャラクターカードをクリック
        └── /characters/[branch]  [1 クリック: 専用ページへ遷移]
              └── 「編集する」クリック
                    └── /characters/[branch]/edit  [+1 クリック]
```

キャラクターだけはクリックで詳細ページへ遷移する（他のコンテンツとの違い）。

#### セッション・シナリオ・キャンペーン・ハウスタブ

```
/profile/[actor]（各タブ）
  └── アイテムをクリック
        ├── デスクトップ: 右サイドペインが開く  [1 クリック]
        └── モバイル: ボトムシートが開く       [1 タップ]
              └── 「編集する」
                    ├── デスクトップ: ペイン内で編集モード切り替え  [+1]
                    └── モバイル: シート内で編集モード切り替え      [+1]
```

シナリオ・ハウスは専用ページを持たない。詳細・編集はすべてペイン / シート内で完結する。

### ハウスルールへのアクセス

```
/profile/[actor]（ハウスタブ）
  └── ハウスをクリック → ペイン Stack[0]: ハウス詳細 [1]
        └── ペイン内の「ハウスルール」セクションを展開 / インライン編集 [+1]
```

ハウスルールはハウスのサブセクションとして同一ペイン / シート内に展開する。専用ページは作らない。

### キャラクターシート型定義へのアクセス

```
/profile/[actor]（シナリオタブ）
  └── シナリオをクリック → ペイン Stack[0]: シナリオ詳細 [1]
        └── ペイン内「キャラシ型定義」セクション
              └── 「型定義を編集 →」 → ペイン Stack[1]: 型定義編集 [+1]
```

型定義は owner のみ Stack[1] へ展開できる。閲覧者は read-only の概要のみ表示。

### 共有 URL と AT URI

AppView は画面共有用 URL と canonical record 参照を分けて扱う。

| 対象 | 画面共有用 URL | canonical record 参照 |
|------|----------------|-----------------------|
| プロフィール | `/profile/[actor]` | `at://[did]/.../player-profile` |
| キャラクター | `/characters/[branch]` | character-branch の `at://` URI |
| セッション | `/profile/[actor]?pane=session:[recordKey]` | session record の `at://` URI |
| シナリオ | `/profile/[actor]?pane=scenario:[recordKey]` | scenario record の `at://` URI |
| キャンペーン | `/profile/[actor]?pane=campaign:[recordKey]` | campaign record の `at://` URI |
| ハウス | `/profile/[actor]?pane=house:[recordKey]` | house record の `at://` URI |

画面共有用 URL は AppView で閲覧するための入口である。`at://` URI は AT Protocol 上の record を正確に参照するための入口である。共有メニューは「リンクをコピー」と「at:// URI をコピー」の両方を表示する。

ペイン内コンテンツの画面共有用 URL は最前面ペインだけを表す。ペインを開いた順番や背面スタックは共有 URL に含めない。

### キャラクター共有

```
/characters/[branch]
  └── 「共有する」ボタン（hero 領域またはオーナーツールパネル）[1]
        └── URL コピーダイアログ or native share sheet（モバイル）
```

### CCFolia 形式でエクスポート

```
/characters/[branch]（owner view）
  └── owner ツールパネル「エクスポート」ボタン [1]
        └── CCFolia 形式のエクスポートダイアログ（ダウンロード or コピー）
```

合計 1 クリックでエクスポート起動。

## 画面別仕様

### `/` — トップページ

詳細仕様は [top-page.md](top-page.md) を正本とする。

---

### `/home` — サインイン後ホーム

**目的**: 継続作業の再開と次のアクションへの最短導線。

```
┌─────────────────────────────────────┐
│  おかえりなさい、[name]             │
├─────────────────────────────────────┤
│  最近のキャラクター                 │
│  [card] [card] [card]               │
├─────────────────────────────────────┤
│  記録が残っていないセッションがあります  │  ← 任意（セッション未記録の場合のみ）
│  「セッションを記録する →」          │
├─────────────────────────────────────┤
│  [＋ 最初のキャラクターを作ろう]    │  ← キャラクターが 0 件の場合のみ
└─────────────────────────────────────┘
```

- キャラクターが 0 件のとき: 空状態 UI + 大きな作成 CTA を表示する
- それ以外: 最近のキャラクターカードを 3〜5 件表示する

---

### `/profile/[actor]` — プロフィールページ

**目的**: そのプレイヤーの自己紹介と、所有する公開コンテンツのハブ。
actor が認証中アカウントの DID または検証済み handle を指す場合は owner mode（draft 含めて表示、編集導線あり）で表示する。他人の actor を指す場合は public mode（public 状態のみ表示、編集導線なし）で表示する。

自分のプロフィール URL を共有しやすくするため、自分・他人の両方を同じ URL 構造に統一する。
左サイドバー / 下部タブの「マイページ」は `/profile/[自分のactor]` へ遷移する。自分の handle が検証できる場合は handle URL を使い、handle が検証できない場合は DID URL を使う。

```
┌──────────────────────────────────────────────────────┐
│  [avatar]  表示名                       [共有する]   │
│             自己紹介テキスト                         │
│             [プロフィールを編集する] (owner only)    │
├──────────────────────────────────────────────────────┤
│  キャラクター  セッション  シナリオ  キャンペーン  ハウス  │
├──────────────────────────────────────────────────────┤
│  [コンテンツ一覧]                                    │
└──────────────────────────────────────────────────────┘
```

**プロフィール編集**:
- デスクトップ: 「プロフィールを編集する」→ `/profile/[自分のactor]/edit`（専用ページ）
- モバイル: 「編集する」→ ボトムシートでフォームを展開

**他人のプロフィールでのタブ動作**:
- 各タブの中身は public 状態の record のみ
- アイテムクリック時の挙動は owner mode と同じ（ペイン / シートで詳細表示）。ただし編集ボタンは出ない
- キャラクタータブからのカードクリックは `/characters/[branch]` 公開ページへ遷移

**各タブの動作**:

| タブ | 一覧の見た目 | クリック時の動作 |
|------|------------|----------------|
| キャラクター | portrait + 名前 + システムのカードグリッド | `/characters/[branch]` へ遷移（ページ移動）|
| セッション | 日付 + シナリオ + ロール + 結果のリスト | 右サイドペイン / ボトムシートで詳細 + 編集 |
| シナリオ | タイトル + システムのリスト | 右サイドペイン / ボトムシートで詳細 + 編集 |
| キャンペーン | 名前 + セッション数のリスト | 右サイドペイン / ボトムシートで詳細 + 編集 |
| ハウス | 名前 + 外部リンクのリスト | 右サイドペイン / ボトムシートで詳細 + 編集 |

キャラクタータブの「下書き」バッジ: draft の character-branch はカードに「下書き」バッジを付けて先頭に配置する。draft を一覧から隠さない（owner は自分の draft を見られる）。

### `/profile/[actor]/edit` — プロフィール編集（デスクトップ）

専用ページで全フィールドを編集する。owner 以外がアクセスした場合は 404 とする。

- セクション: 基本情報（displayName, description, avatar, banner）
- セクション: TRPG プロフィール（主な役割, プレイ形式, 使用ツール, 所持ルールブック, 好みのシナリオ, プレイスタイル, 地雷・苦手, できること・スキル, プレイ可能時間帯）
- 保存・キャンセルボタン（ページ上部 sticky）

モバイルでは `/profile/[actor]/edit` へ遷移せず、`/profile/[actor]` から直接ボトムシートを開く。

### `/characters/new` — キャラクター作成

ステップ形式のウィザードページ。

```
ステップ 1: ルールシステムを選ぶ（rulesetNsid の選択）
ステップ 2: キャラクターシート型を選ぶ（schema 選択）
ステップ 3: フィールドを入力する（schema の fieldDefs に基づく動的フォーム）
ステップ 4: 立ち絵を設定する（任意）
ステップ 5: 公開設定を選ぶ（draft / public）
```

シナリオ経由の作成（scenario に recommendedSheetSchemaRef がある場合）:
- ステップ 1・2 をスキップし、ステップ 3 から開始する

作成完了後 → `/characters/[branch]`（新規作成した詳細ページ）へ遷移。

## ルートツリー（確定版）

| route | surface | 認証 |
|-------|---------|------|
| `/` | public top | public |
| `/home` | signed-in home | owner |
| `/profile/[actor]` | プロフィールページ。`actor` は DID または handle | public + owner |
| `/profile/[actor]/edit` | プロフィール編集（自分のみ。デスクトップのみ遷移） | owner |
| `/characters/new` | キャラクター作成ウィザード | owner |
| `/characters/[branch]` | キャラクター詳細（canonical shared） | public + owner |
| `/characters/[branch]/edit` | キャラクター編集 | owner |

**専用ルートを持たない（サイドペイン / ボトムシートで完結）**:
- セッション 詳細・作成・編集
- シナリオ 詳細・作成・編集
- キャンペーン 詳細・作成・編集
- ハウス 詳細・作成・編集（ハウスルール含む）

**route として定義しない path**:
- `/profile`
- `/profile/edit`
- `/players/[did]`
- `/scenarios`
- `/scenarios/[scenario]`
- `/scenarios/[scenario]/schema`
- `/houses/[house]`
- `/houses/[house]/rules`

## Apple UX 対応方針

| 原則 | 本設計での実現 |
|------|--------------|
| 直接性 | 主要アクション（キャラクター作成）は 1 クリックで到達 |
| 文脈の維持 | サイドペイン / ボトムシートで詳細を開いても一覧コンテキストを失わない |
| 段階的開示 | ボトムシートは peek → full の 2 段階。作成フォームはステップ形式 |
| 一貫性 | デスクトップとモバイルで同一の概念が同一の位置にある |
| 即時フィードバック | 保存操作はペイン / シート内で完結し、結果を即表示する |
| 最小限の chrome | サイドバー・タブバーはコンテンツを圧迫しない幅と視覚重みで設計する |
| スワイプ操作 | モバイルでは作成モーダル内のページ切り替えをスワイプで行う |
| モーダル階層の浅さ | ボトムシート内からさらにモーダルを開かない。必要ならページ遷移に切り替える |

## 依存グラフ

```text
Identity resolution
  ├─ /profile/[actor]
  │   ├─ owner mode / public mode 判定
  │   ├─ プロフィールタブ表示
  │   │   ├─ キャラクタータブ → /characters/[branch]
  │   │   ├─ セッションタブ → pane=session:[recordKey]
  │   │   ├─ シナリオタブ → pane=scenario:[recordKey]
  │   │   ├─ キャンペーンタブ → pane=campaign:[recordKey]
  │   │   └─ ハウスタブ → pane=house:[recordKey]
  │   └─ プロフィール共有 URL
  └─ handle invalid state
      └─ DID URL への fallback 表示

Pane state
  ├─ URL query は最前面 pane だけを保持
  ├─ 背面 stack はページ内状態として保持
  ├─ AppView URL 共有
  └─ at:// URI 共有

Character shared root
  ├─ /characters/[branch]
  ├─ character detail first view
  ├─ public-safe session history embed
  └─ CCFolia export / owner tools
```
